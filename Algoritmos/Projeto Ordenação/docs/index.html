Algoritmos Otimizados

Além dos algoritmos clássicos, o projeto inclui versões otimizadas (use --optimized nos benchmarks). Fontes das otimizações:

  • Bubble Sort (Optimized) — Early exit: interrompe quando nenhuma troca ocorrer na passada. O(n²) pior caso, O(n) melhor. Otimização clássica da literatura de algoritmos.
  • Insertion Sort (Sentinel) — Menor elemento na posição 0 elimina a verificação j >= 0 no loop interno. Cormen et al., Introduction to Algorithms, Cap. 2.
  • Selection Sort (Double) — Busca menor e maior por passagem, posicionando ambos nas extremidades. Reduz iterações à metade. Variação conhecida na literatura de ordenação.
  • Merge Sort (Single Aux Array) — Um único array auxiliar para todo o merge, alocado uma vez. Cormen et al., Introduction to Algorithms, Cap. 2.
  • Quick Sort (Hoare + Median + Insertion) — Particionamento de Hoare (Cormen, Cap. 7), pivô mediana-de-três (Sedgewick), Insertion Sort para subarrays pequenos e eliminação de recursão de cauda (padrão em implementações híbridas).
